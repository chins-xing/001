# 宠物百科项目 - AI开发提示词

## 一、项目概述

构建一个宠物百科Web应用，包含宠物品种介绍、喂养知识、洗护教程、萌宠图集和数据大屏功能。采用前后端分离架构，前端负责展示，后端负责数据管理和搜索服务。

---

## 二、功能需求

### 2.1 前端页面（共3个页面）

#### 页面1：首页（index.html）
- 展示宠物品种卡片列表，使用Flex布局
- 卡片包含：宠物头像、品种名称、简介
- CSS3实现宠物头像动效（如呼吸动画、旋转等）
- 卡片悬浮过渡特效（hover时放大、阴影变化等）
- 萌宠图集轮播组件（JS实现自动/手动切换）
- 搜索框（唯一与后端交互的功能）

#### 页面2：百科页（wiki.html）
- 喂养知识展示
- 洗护教程步骤切换（JS实现分步展示）
- 品种详情内容
- 搜索框（与后端交互）

#### 页面3：数据大屏（dashboard.html）
- 宠物品种分布图表（饼图/柱状图）
- 养护打卡统计图表
- 健康指标展示
- 宠物成长趋势图表（折线图）
- 支持按宠物类型筛选数据
- 使用虚拟数据展示

### 2.2 后端服务

#### Python服务（数据处理与搜索）
- 读取JSON数据文件
- 提供搜索API接口
- 处理数据查询请求
- 支持配置文件热重载

#### Go服务（数据流处理）
- 作为数据网关/代理
- 处理前端请求转发
- 数据流管理
- 支持配置文件热重载

## 三、架构设计约束

### 3.1 前后端分离
- 前端仅通过搜索功能与后端交互
- 前端整体消费来自后端的数据流
- 其他页面内容从本地JSON文件读取

### 3.2 配置管理（关键约束）
- **禁止任何硬编码参数**
- 所有配置通过JSON配置文件读取
- 配置文件包括但不限于：
  - 服务端口配置
  - API地址配置
  - 数据文件路径配置
  - 图片路径配置
  - 页面展示参数配置
- **所有配置文件必须支持热重载**（修改后无需重启服务即可生效）

### 3.3 数据存储
- 所有文本数据使用JSON文件存储在本地
- 文本数据与图片必须建立明确关联
- 关联表使用JSON配置文件管理
- 数据结构示例：
```json
{
  "pets": [
    {
      "id": "pet_001",
      "name": "金毛犬",
      "category": "狗",
      "image": "images/pets/golden_retriever.jpg",
      "description": "性格温顺，聪明忠诚...",
      "feeding_guide": "feeding/golden_retriever.json",
      "grooming_guide": "grooming/golden_retriever.json"
    }
  ]
}
```

## 四、技术栈要求

### 4.1 前端
- HTML5 + CSS3 + JavaScript（原生或轻量框架）
- Flex布局实现响应式卡片陈列
- CSS3动画实现头像动效和过渡特效
- JS实现图集轮播和教程步骤切换
- 数据大屏使用图表库（如ECharts、Chart.js）

### 4.2 后端
- Python：数据处理、搜索服务（可使用Flask/FastAPI）
- Go：数据流处理、请求转发
- 两者均需实现配置文件热重载机制

## 六、开发注意事项

### 6.1 配置热重载实现要点
- Python：使用`watchdog`库监控文件变化，或使用定时轮询
- Go：使用`fsnotify`库监控文件变化
- 前端：可通过定时fetch或使用`localStorage`事件监听

### 6.2 数据与图片关联
- 必须通过`image_mapping.json`维护关联关系
- 图片路径变更时只需修改配置文件，无需改代码
- 支持图片CDN路径配置

### 6.3 搜索功能
- 前端搜索框发送请求到后端
- 后端返回匹配的宠物品种、知识、教程
- 搜索结果在前端高亮展示

### 6.4 数据大屏
- 使用虚拟数据，但数据结构需与真实数据格式一致
- 图表需支持动态刷新
- 筛选功能通过前端JS实现（数据量小）或后端API（数据量大）
